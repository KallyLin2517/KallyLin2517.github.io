《亲密关系》：寻找自我是一场格外漫长的旅行

2020-03-15 读书笔记

书籍详情

书名：《亲密关系》

作者：【加】克里斯多福·孟

阅读途径：实体书/上网自己找资源/百度阅读/其他阅读器

正文

以前看书的时候，有时会困惑阅读是为了什么。但所幸人是能慢慢成长的，慢慢的我也好像明白了这个问题的答案——

大约是为了寻找答案吧。

人会产生各种各样的困惑，随着长大经历的事情越多，问题可能也会越积愈多。

我对这个世界知之甚少，对我自己也弄不清楚，弄不明白。为此我一直在寻找某些问题的答案，为何而伤心，为何而开心，又为何而痛苦。

寻找的一开始或是希望能远离某些不好的情绪，当时的想法也挺天真，如果弄清楚负面情绪的成分后，是否就能自己去分解它了呢？

哈哈。

阅读传统小说也好，网络小说也好，其他体裁的故事也好，都不过是为了找到问题的答案罢了。很多问题的答案不会明明白白告之阅读的人，但我可以通过字里行间窥见作者的内心世界，或许就可以在作者的世界中探寻到我所想要的答案。

这么一描述，忽然就有种探险家的感觉啊，听起来真厉害。

会接触《亲密阅读》这本书，也起源于我内心一个始终未能解开的困惑。

大部分时候，我不觉得自己和其他人有多大区别。但是在很多个偶然的时刻，我会忽然触及到自己内心的冷漠。

是的，没什么好否认的。无论我外在表现如何，很多时候，我内心毫无波澜。

我发现自己这个问题那天，一个朋友发烧了，状态很不好。我第一反应并不是“担心”这种情绪，而是“她生病了，我应该安慰她”这个冷冰冰的想法。

好吧，更诚实一点，不要说第一反应了，我根本全程无法产生“担心”这种情绪。

大约从那时开始，我就开始厌恶自身这个特点，厌恶起了自身的存在。

由此开始审视我的过去。小学，初中，高中，每一年每一年学校就要换一次班级。我会在那一年和周遭的人相处得好，但是一年过去，过往就仿佛被抹去一般，人事物都难以在我心里留下痕迹——我也不想留。

哈哈，其实这么说也不太对，至少很多我讨厌的回忆我虽然记不清，却依旧给我造成了很大影响。

另一个给我感触颇深的结点，还是在高三那年。我认为自己与他人一样，平平无奇，既不至于让人多么讨厌，却也没有什么值得人喜欢的点。但是当时遇到一个朋友，她热情得莫名其妙，对我的好是直接而明媚。

当时我认为所有来得忽然的情感都是轻浮而短暂的，如水面掠影，转瞬之间归于虚无。所以我起初并不是很相信她，我以为她对谁都这样。我有点烦她，认为她进军我内心领地的行为，是触及了我的底线。

……我应该庆幸自己还没有那么丧心病狂，最后才没错过这个朋友。

某天，她也终于发现了我的真实面目，开始变得沉默。那天我开始良心不安，仔细想一想就开始发现自己的行为确实过分了，这都是什么跟什么啊。

正巧那天宿舍熄灯后，我还听见她在自己的床里好像一边写作业一边在抽泣。当我想要去安慰她时，我感受到了深深的恐惧。我不知它从哪来，为什么会产生，总之那一刻我十分害怕。当时脑中的想法已经不记得了，大约是“赶紧睡吧，没必要的，别管了”之类的吧。

感激过去的一点是，我当时克服了恐惧，咬牙下了床去了她的床上找她。

哈哈，幸好啊……

看《亲密关系》时，作者提及一句话，大致就是：当你面临改变时，你的内心会有一个声音阻止你去改变。当这个改变对你的人生影响越重大，阻止的声音也就越强烈。

当时看到这句话，我就想起了我和这个朋友的过往，想起了堪称我们俩关系转折点的那个晚上。在那之后，我们才能真正称为“朋友”，而非我过一年就抹去痕迹的“某某人”。

直到现在，我还是很感激自己当时做出的选择，也发自内心喜欢而感激我这个朋友。因为在她身上，我学到了很多我所缺失的东西。

如一往无前的勇气，如不准许被拘束的自由，又如，火焰般张扬明媚的情感。

她让我明白，友善也好热情也好，是要分人的。

尤其是，假如名为情感的杯子里装的爱本来就不多，必须，必须十分慎重地分给别人——去分给自己珍视的人。假如不知道自己珍视的人，那就去分给珍视自己的人。因为本身已经没有其他能给的东西，只能把这些作为谢礼了。

我是如此想着，可常常发现，就连这样也很难做到。

靠近，这一个动作常常使我不安而惶恐，我却对我自己的情绪感到不解。

说真的，我在怕什么？在我已经忘记的东西里，真的有什么东西让我如此害怕着吗？

虽然一开始我并不是因为这个问题翻开《亲密关系》这本书的，可我却在这本书，隐隐约约好像看到了这个问题的答案。

是的，应该用“意外之喜”来形容我看完这本书的心情吧。

亲密关系，包括而不仅限于爱情。友情、亲情等等其他一切接近接触的情感，都应该能算是亲密关系。

而这本书与其说它在讲如何处理好亲密关系，不如说它是在告之读者如何在亲密关系中发现自我，探寻到完整的自我。

正如书的一开头所说，寻找真挚永恒的亲密关系，其实就是在寻找自我。

骑士为什么找不到他的圣杯？因为圣杯就在他自己心中，而他始终不相信这一点。

作者并不是单纯告诉人们如何去维持一段关系，而是告诉你，无论一段关系是好是坏，结局走向如何，都不是最重要的。

最重要的是，该如何在与每一个人的相处中，剥去喧哗繁杂的表象，洗尽铅华，找寻到真正的自我。

对于真实自我的探寻，我认为才是这本书真正的主题。

自我，是什么呢。

或许在先天基因和后天环境共同作用下产生的综合体，即我能感知到的自我。

在各色各样的环境下下成长的人啊，很经常的，会丢失自己某种东西。就像壁虎为了逃命，自断其尾。

这件事有点意思，不知道应该说是聪明的举动还是愚蠢的行为。

多年之前，曾在橙光玩过一个游戏。遭受校园暴力的主人公为了不被霸凌者想起，努力让自己变得没有存在感；为了使得霸凌者感觉欺负她很无聊，努力变得呆板无趣。

讽刺的是，唯有这样的选择，游戏才能继续下去。选择反抗，最后会得到一个BE。

当主人公离开了噩梦般的旧学校后，来到新学校的她以为能开始新的生活。却发现因为之前学会的“没有存在感”让别人都注意不到她，“呆板无趣”让别人不想和她聊天。

她丢弃的“存在感”“有趣”，在校园欺凌阶段保护了她，却在环境变化时就摇身一变成了她的囚牢。

那是一个让我感触颇深的游戏，见证了主人公从快乐到绝望再到自我救赎的过程，我的心情变化也是十分大的。

我想很多人都一样，成长途中，为了保护自己，会像壁虎一样不得已主动丢失自己的某一部分。

而丢失的这部分，长大后，有些人会很渴望找回来。

起初我并没有想这么多，在翻了这本书后我才突然被作者告知，我居然还有“找回来”这样一个选项。

当时的心情有点不可思议，而找回来的方法就在人与人的相处之中。

在每一段渐近的关系中，相似的情景会触动内心的旧伤，曾经抛弃某一部分的场景会再次显现。乍一听似乎是很不好的事情，可作者告诉我的，说这是一个机会。一个重新抉择，重新面对过往的机会。

你可以抉择不要放弃自己的某一部分，可以选择以正确的姿态终结这段阴影，疗愈自己。现在是现在，过去是过去，你已经不一样，已经有能力去改变。

就这样，在一个个重新浮现的旧伤里，像时空旅行者回到过去一般，去重新抉择那些影响一生的选择。

我就这样敲着这些文字，文字敲出来并不困难，不过是手指之间的起起落落。可要做到这些，怕是现在的我根本无法想象的。

所以说，寻找自我，是一场格外格外漫长的旅行。它漫长到要用一生来完成。漫长到就算究其一生去探寻，也有可能完全找不到终点。

不过我想，就算这样也没有关系吧。在茫茫人生长河中，人能不为了单纯活着而活着，或者是一种幸运，而我不想浪费这种幸运。

我时常还会想起那个朋友，她去年高考失利，今年在复读。我挺想她的。

我时常觉得遇到她已经花掉我九成的运气，所以这运气怎么能白白浪费，我希望自己永远不要忘记和她在一起时学会的东西。

即情感。她那样无所畏惧，自信张扬，简单纯粹对我好，是真的曾给予我惊艳，点亮了我世界。

虽然，我还是很难做到她那样，但我会努力的。

java·四则混合运算计算器

2020-03-14 Java

前言

刚学java时，被要求一段时间内做出一个计算器。当时一点思路都没有，上网找了不少教程不过看不懂。幸好，难题是慢慢被解决的，这个困扰了我很久的问题现在已经解决，于是来记录一下。仅以我自己的角度，尽可能写详细。

需要具备的知识点

  • 需要有java基础，入门教程可参考慕课网java入门第一季、第二季、第三季。个人认为如果只是做计算器，学到第三季第三章就差不多了。

  • 数据结构关于栈的知识。只做计算器的话，知道java的栈用法就可以了，不过尽可能还是系统学习下数据结构的知识。因为只是单学栈，真的很容易一脸懵逼，而且也没有多大用处。数据结构推荐一下小甲鱼的《数据结构与算法》 （这个是C语言版），虽然频繁开车，但是作者语言轻松幽默，学起来轻松一点。

  • 正则表达式。正则表达式是个好东西，做计算器虽然也不要求懂那么深，但还是要初步了解一下。

能够实现的功能

目前的java计算器是控制台版本，输入一串算式并以“=”号结尾后就可以计算最终结果。支持整数与小数混合四则运算，不支持“（）”。我测试时最大的结果可以达到十的几百次方（最大可能是三百多），再往上，它结果就会显示“无穷”。有异常处理，非法输入会得到提醒。

运行结果如下：

请输入算式(以“=”结束,不允许添加“（）”)：

22.4+8*2-1+9/3=

运算结果为：40.4

其中下划线的内容是可输入的。

实现思路

  1. 建立两个栈，分别用来存储运算符和数字；
  2. 用一个字符串存储从键盘键入的算式；
  3. 用正则表达式处理原始字符串，按顺序将数字和运算符存入两个数组中。如“1+9/3-2=”中，处理后得到数字数组num[4]={1,9,3,2},得到运算符数组char[4]={+,/,-,=}。
  4. 数字不需要判断优先级，运算符需要，所以要为运算符设定等级。这里假设”=”的等级为1，”+ -“等级为2，“* /”等级为3。当即将入栈的运算符等级等于或小于栈顶元素的运算符等级，则需弹数字栈两次，弹运算符栈一次。
  5. 首先让num[0]入数字栈，然后取char[0],比较它与栈顶元素的等级。因此此时运算符栈为空栈，无需判断直接入栈。再取num[1]放入数字栈，接着取char[1]。因为”/“的等级大于此时运算符栈的栈顶元素等级，继续压栈。第三回取num[2]放入数字栈，再取char[2]。此时的char[2]是“-”，等级为2，小于运算符栈的栈顶元素，因此不能压栈，先把它放一边。对运算符栈弹栈一次，对数字栈弹栈两次。这里弹出来的数字是9和3，弹出来的运算符是/，将9和3进行除法计算，将结果入栈。此时数字栈内是3和1，运算符栈是“+”。把之前扔一边的char[2]再取过来，继续和栈顶元素比较，发现二者等级相等，依旧不能入栈，而是要弹数字栈两次，弹符号栈一次进行计算。1+3结果为4，将4压入数字栈，第三次把char[2]取过来。因为此时运算符栈已经是空栈，于是直接让char[2]压栈即可。后面运算过程就是如此，直到最后运算符栈只剩下一个“=”的时候，数字栈也只剩下一个元素，就是最终得数。

请不要责怪我排版为何如此糟糕，我不太会用Markdown语言orz

我觉得第三步和第五步是最重要的，只要明白了如何用栈处理混合四则运算式，然后用正则表达式帮助切割输入的字符串，问题基本就解决了。

代码实现

package com.calculator;//这个是包名

//导入各种东西

import java.util.Arrays;

import java.util.EmptyStackException;

import java.util.InputMismatchException;

import java.util.Scanner;

import java.util.Stack;

public class Intital {

public double c;

//计算

public double count(double b,double a,char ch) { 

   switch(ch) {

   case '+':

       c=a+b;

       break;

   case '-':

       c=a-b;

       break;

   case '*':

       c=a*b;

       break;

   case '/':

       if(b==0) {

           throw new ArithmeticException(); 

       }

       c=a/b;

       break;

   }

   return c;

}

public static void main(String[] args) {

   try {

   Intital test = new Intital();

   //数字栈

   Stack stackNum=new Stack();

   //运算符栈

   Stack stackChar=new Stack();

   //运算符等级数组

   int[] rank = new int[15];

   Scanner input = new Scanner(System.in);

   System.out.println("请输入算式(以“=”结束,不允许添加“（）”)：");

   String str0 = input.nextLine();

   //int tempRank = 0;

   //存储数字

   String[] str1 = str0.split("\\+|\\-|\\*|\\/|\\=");    


   //存储运算符

   String str2 = str0.replaceAll("[^\\\\*|\\\\/|\\\\+|\\\\=|\\\\-]", "");

   //将str2转换为字符串数组

   String[] str3 = str2.split("");

   //至此，str1[]存储着数字，str3[]存储着运算符

   //获取str1的长度

   int str1Length = str1.length;

   for(int i=0;i<str1Length;i++) {

       //将字符转为double型

       double num = Double.parseDouble(str1[i]);

       //数字进栈

       stackNum.push(num);

       switch(str3[i].charAt(0)) {

       //"="的等级为1，"+ -"等级为2，“* /”等级为3

       case '+':

       case '-':

           rank[i]=2;

           break;

       case '*':

       case '/':

           rank[i]=3;

           break;

       case '=':

           rank[i]=1;

           break;

       }

       int j = i-1;

       //判断是否空栈,如果是直接压栈;如果不是空栈，需要判断优先级

       if(stackChar.empty()) {

           stackChar.push(str3[i].charAt(0));

       }else {

           //当strack2不为空时，分为两种情况，

           //rank[i]>rank[j]和rank[i]<=rank[j]

           //当rank[i]<=rank[j]时一直循环

           while(rank[i]<=rank[j]) {

           double total = test.count((double)stackNum.pop(), (double)stackNum.pop(), (char)stackChar.pop());

            j--;

            stackNum.push(total);

            //在这里需要判断一下运算符栈是否为空，为空则停止循
            环，跳出

            if(stackChar.empty()) {

               break;

            }

           }

           stackChar.push(str3[i].charAt(0));

       }

   }

   System.out.println("运算结果为："+stackNum.pop());

   } catch (NumberFormatException e) {

       System.out.println("请输入正确的运算式,或首位数不要为负数。");

   } catch (ArithmeticException e) {

       System.out.println("除数不能为零。");

   } catch (ArrayIndexOutOfBoundsException e) {

       System.out.println("请以“=”结束。");

   } catch(EmptyStackException e) {

       System.out.println("请输入正确的运算式，或首位数不要为负数。");

          }
   }}
